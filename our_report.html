1 I. Pipeline d’extraction des données

Les données sont récupérées depuis les portails open data français (opendata.paris.fr, data.gouv.fr). On s’appuie sur un fichier informations_dataset.csv qui centralise les métadonnées de chaque jeu de données : l’URL de téléchargement, le nom du fichier et son extension. Le téléchargement se fait via le package httr, et on stocke tout en local dans un dossier dédié.

2 II. Pipeline de nettoyage des données

Avant de pouvoir analyser quoi que ce soit, il faut mettre les données en forme. Les fichiers bruts téléchargés sont hétérogènes : formats différents, noms de colonnes incohérents d’une élection à l’autre, valeurs manquantes.

On applique les étapes suivantes :

  • Conversion des fichiers .txt / .csv en .parquet (plus léger et plus rapide à charger).
  • Suppression des lignes avec des valeurs manquantes.
  • Renommage des colonnes pour avoir un schéma cohérent sur tous les jeux de données (code_dep, code_com, bureau_vote).
  • Passage en format wide : une ligne = un bureau de vote, une colonne = un candidat ou une liste.

Un point d’attention : certains partis ont changé de nom au fil des années : le FN est devenu le RN en 2018, plusieurs mouvements de gauche se sont regroupés sous différentes étiquettes. Ces changements de nuances compliquent les comparaisons inter-élections, en particulier pour la partie régression.

3 III. Factorisation matricielle (ACP / SVD)

Pour chaque tour d’élection, les données se présentent sous forme d’un tableau où les lignes sont les bureaux de vote et les colonnes le nombre de voix obtenus par chaque liste ou candidat. On centre et réduit ce tableau avant d’appliquer une ACP : ce qui revient à faire une SVD de la matrice centrée-réduite : pour identifier les principaux axes de variation politique entre les bureaux.

3.1 ACP, Européennes 2009 (Paris)

On commence par les européennes de 2009 à Paris. Après centrage-réduction, on applique prcomp() sur la matrice des votes.

PC1 concentre la majeure partie de la variance : elle capte le gradient politique dominant entre les bureaux de vote parisiens. Les composantes suivantes apportent chacune une part nettement moindre. PC1 explique à elle seule 51% de la variance totale contre 6,3% pour PC2. On peut donc se contenter des premières composantes pour résumer les données.

Presque toutes les flèches pointent vers la gauche sur Dim1 avec des contributions élevées (en orange). On voit deux groupes de listes qui s’opposent sur Dim2 : certaines tirent vers le haut, d’autres vers le bas. Les listes proches du centre contribuent peu et ne structurent pas vraiment les données.

La plupart des bureaux se concentrent dans la zone centrale du graphique : leurs profils se ressemblent assez. Quelques bureaux s’écartent fortement vers des valeurs négatives de PC1, ce sont des bureaux où une liste a recueilli bien plus de voix que la moyenne. Les points en orange (cos2 élevé) sont bien représentés dans ce plan.

3.2 ACP : Européennes 2024 (Paris)

On refait la même analyse pour les européennes de 2024. Le paysage politique a pas mal évolué en quinze ans, montée en puissance du RN, recomposition à gauche, ce qui devrait se refléter dans la structure des composantes.

En 2024, PC1 n’explique plus que 15,2% contre 51% en 2009. Cela s’explique surtout par le nombre bien plus grand de listes en 2024, qui répartit la variance sur beaucoup plus de dimensions. Il faut donc plus de composantes pour résumer les données qu’en 2009.

Le graphique est plus chargé qu’en 2009 du fait du nombre de listes. Sur Dim1, on distingue deux groupes opposés : à gauche bardella_jordan et marechal_marion, à droite glucksmann_raphael et toussaint_marie. Sur Dim2, hayer_valerie se démarque clairement vers le haut.

Le nuage de points est plus compact qu’en 2009, la plupart des bureaux étant proches de l’origine. Le cos2 souvent faible confirme que deux composantes ne suffisent pas à bien représenter tous les bureaux, ce qui est cohérent avec la variance expliquée plus faible.

3.3 Comparaison 2009 / 2024 par SVD inter-élections

Pour comparer directement les structures de vote entre 2009 et 2024, on calcule la matrice de covariance croisée entre les deux tableaux centrés-réduits, restreints aux bureaux présents dans les deux élections, puis on en fait la SVD. Les valeurs singulières mesurent dans quelle mesure les deux élections co-varient dans chaque direction, autrement dit, si les bureaux qui votaient d’une certaine façon en 2009 continuent à le faire en 2024.

La corrélation élevée entre les premiers variés canoniques montre que le principal axe de variation politique à Paris est resté stable entre 2009 et 2024, malgré les recompositions partisanes. Les quelques bureaux qui s’écartent de la droite rouge sont ceux dont le profil a le plus changé entre les deux élections.

4 IV. Clustering

On cherche maintenant à regrouper les bureaux de vote selon leur profil de vote. On travaille sur les coordonnées des 5 premières composantes principales issues de l’ACP sur les européennes 2024, ce qui permet de réduire le bruit tout en conservant l’essentiel de l’information.

4.1 Choix du nombre de clusters

La courbe montre un coude assez net : au-delà d’un certain k, le gain en inertie intra-classe devient marginal. On confirme ce choix avec le critère de silhouette. Le coude se situe à k=4 : avant ce point l’inertie baisse vite, après elle se stabilise et augmenter k n’apporte pas grand chose.

4.2 Critère de silhouette

La silhouette identifie k = 2 comme la partition la plus cohérente.

4.3 K-means

Les 4 clusters sont bien visibles dans le plan PC1/PC2. Les clusters 1 et 2 se chevauchent un peu à droite de PC1. Le cluster 4 regroupe des bureaux plus à gauche sur PC1. Le cluster 3 est un seul point très écarté : c’est un bureau dont le profil ne ressemble à aucun autre.

4.6 Comparaison des méthodes de clustering

En comparant les silhouettes moyennes pour k=4, on obtient : K-means (0,499), CAH lien complet (0,488), CAH Ward D2 (0,441) et CAH lien moyen (0,433). Le K-means donne la meilleure partition, suivi de près par le lien complet. Ward D2 et lien moyen sont un peu en dessous. Dans tous les cas, des valeurs entre 0,44 et 0,50 indiquent que les clusters sont raisonnablement bien séparés.

5 V. Régression linéaire

On cherche à prédire les résultats du premier tour des législatives 2022 à partir des scores des législatives 2017, bureau de vote par bureau de vote. L’idée est simple : dans quelle mesure les scores de 2017 permettent-ils d’anticiper ceux de 2022 ? On ajuste une régression linéaire ordinaire (OLS) par nuance présente en 2022, en utilisant comme variables explicatives les pourcentages de voix de chaque nuance en 2017.

Pour chaque nuance présente en 2022, on ajuste un modèle linéaire sur 80 % des bureaux et on évalue les prédictions sur les 20 % restants. Les R² sont globalement assez élevés pour les nuances qui existaient déjà en 2017 sous une étiquette proche (RN/FN, LFI/FG, LR…), ce qui montre que les scores passés ont un fort pouvoir prédictif. Pour les nuances nouvelles ou très marginales, la prédiction est logiquement moins bonne.

5.3 Visualisation des coefficients

Pour les 5 nuances les mieux prédites, on regarde quels candidats de 2009 ont des coefficients significatifs (p < 0,05). Pour glucksmann_raphael, les prédicteurs les plus forts sont elisabeth_barbay et michel_barnier. Pour hayer_valerie, alain_mourguy ressort nettement avec un coefficient positif élevé. Pour bellamy_francois_xavier, les coefficients sont plus faibles mais sabine_herold se distingue. Pour toussaint_marie, rolande_perlican a un coefficient très élevé (autour de 30-40), ce qui est à prendre avec précaution vu la largeur de son intervalle de confiance. Globalement, les candidats de 2009 les plus prédictifs pour 2024 appartiennent aux mêmes familles politiques, ce qui confirme la stabilité des profils de vote par bureau.

5.4 Visualisation prédit vs observé

Pour les nuances les mieux prédites, les points s’alignent bien sur la diagonale. Les résidus restent néanmoins non négligeables dans certains bureaux, ce qui suggère qu’une partie des dynamiques locales échappe aux seuls scores de 2017, probablement des effets de candidats ou de recompositions locales difficiles à capter avec des nuances agrégées.

5.5 Diagnostics du modèle

Les graphiques de diagnostic montrent que le modèle fonctionne correctement mais n’est pas parfait. Le Q-Q plot montre que les résidus s’écartent un peu de la normale dans les queues : il y a plus de gros résidus que prévu. Le graphique Residuals vs Fitted montre une légère hétéroscédasticité, la dispersion augmentant avec les valeurs ajustées. Les observations 414, 203 et 197 ressortent comme les plus influentes sur la distance de Cook. Ces légères violations des hypothèses sont courantes sur des données de vote agrégées et ne remettent pas en cause les grandes tendances du modèle.

set.seed(42)

test  <- df_reg[-idx, ]

rmse <- function(obs, pred) sqrt(mean((obs - pred)^2))
r2   <- function(obs, pred) 1 - sum((obs - pred)^2) / sum((obs - mean(obs))^2)

resultats <- map_dfr(nuances_22, function(nuance) {
  if (!nuance %in% colnames(train)) return(NULL)
  y_train <- train[[nuance]]
  y_test  <- test[[nuance]]
  if (sd(y_train, na.rm = TRUE) == 0) return(NULL)

  formula_str <- paste0("`", nuance, "` ~ ",
                        paste0("`", predictors, "`", collapse = " + "))
  mod  <- lm(as.formula(formula_str), data = train)
  pred <- predict(mod, newdata = test)

  data.frame(
    nuance = nuance,
    RMSE   = round(rmse(y_test, pred), 3),
    R2     = round(r2(y_test, pred),   3)
  )
})

resultats

  filter(!is.na(R2)) |>
  arrange(desc(R2)) |>

  pull(nuance)
